# 코드베이스 버그 감사 보고서

- **일자**: 2026-06-11
- **대상 버전**: 0.6.21 (`Cargo.toml`)
- **범위**: `src/` 전체 약 88,466줄 / 8개 영역 + `telegram.rs`(21,850줄) 4분할 = 12개 영역 감사
- **방법**: 영역별 정적 분석으로 후보 수집 → 각 후보를 실제 코드(및 `ratatui` 0.28.1 / `russh` 0.46 벤더 소스, 로컬 `rsync` 3.2.7)로 직접 검증 → 확정된 것만 수정
- **결과**: 치명적/중대 버그 **19건 수정**, informational 3건은 비치명으로 미수정

> 빌드는 프로젝트 규칙에 따라 실행하지 않았습니다. 모든 수정의 타입·문법·기존 테스트 영향을 검토했으나 `cargo build`로 최종 확인이 필요합니다. `OnceLock` 사용 1곳(Rust 1.70+ 필요).

## 1. 런타임 패닉 (CRITICAL) — 7건

### 1.1 에디터: 줄 단축 후 커서 미보정 → `Vec::insert`/`remove` OOB 패닉 (3건)
- **파일/함수**: `src/ui/file_editor.rs` — `outdent`(선택 분기), `toggle_comment`, `replace_all`
- **원인**: `clamp_cursor()`가 undo/redo에서만 호출됨. 줄을 짧게 만드는 연산 뒤 `cursor_col`이 그대로 남아, 다음 편집(`insert_char`의 `chars.insert(cursor_col, c)` 등)에서 `index > len` 패닉. `outdent`의 비선택 분기는 `saturating_sub`로 보정하지만 선택 분기는 누락(비대칭).
- **재현**: `// hello`(끝 커서) → Ctrl+/ 로 주석 해제 → 한 글자 입력 → 패닉. 선택 영역 Shift+Tab/치환에서도 동일.
- **수정**: 세 함수 끝에 `self.clamp_cursor()` 호출 추가.

### 1.2 AI 화면: `delete_word_left` 바이트/char 인덱스 혼동 → 슬라이스 OOB 패닉
- **파일**: `src/ui/ai_screen.rs` `delete_word_left`
- **원인**: `trimmed.rfind(' ')`는 **바이트** 오프셋인데 이를 char 인덱스로 `chars[..new_col]`에 사용.
- **재현**: AI 입력창에 `안녕 세상` 입력 후 Ctrl+W → `rfind`가 바이트 6 반환 → `chars[..7]`을 5개 슬라이스에 적용 → 패닉.
- **수정**: `rfind` 제거, char 단위로 후행 공백·단어를 역방향 스캔(동작 동일, char-safe).

### 1.3 Remote-connect 다이얼로그: 바이트 길이를 char 커서로 사용 → 패닉
- **파일**: `src/ui/dialogs.rs` `handle_remote_connect_input` (+ `app.rs` `active_field_value`)
- **원인**: `cursor_pos`는 char 인덱스인데 `active_field_value().len()`(바이트)로 설정/비교(Tab/Down/BackTab/Up/End/Delete/Right 7곳). End/Tab 후 Backspace/Delete 시 `chars.remove(byte_offset)` 패닉.
- **재현**: Host 필드에 `한글` 입력 → Delete 또는 End→Backspace → 패닉.
- **수정**: 7곳 모두 `.chars().count()`로 교체. 비밀번호 마스킹 `"*".repeat(value.len())`도 char 단위로(표시 길이 정확화).

### 1.4 stale `diff_first_panel` 인덱스 → `panels[first]` OOB 패닉
- **파일**: `src/ui/app.rs` `close_panel` / `start_diff`
- **원인**: `close_panel`이 `active_panel_index`·AI 인덱스는 보정하나 `diff_first_panel`은 누락. diff 첫 선택 후 패널을 닫으면 stale 인덱스로 `self.panels[first]` 접근.

- **수정**: `close_panel`에 `ai_previous_panel`과 동일한 인덱스 시프트/무효화 추가. `start_diff`에 `.filter(|&f| f < self.panels.len())` 방어 가드 추가.

### 1.5 작은 터미널에서 dialog가 버퍼 초과 → ratatui `Clear` OOB 패닉 (3곳)

## 3. 데이터 유실 / OOM / 보안 — 6건

### 3.1 설정 파싱 실패 시 종료할 때 기본값으로 덮어써 사용자 설정 영구 삭제
- **파일**: `src/main.rs`
- **원인**: `~/.cokacdir/settings.json` 파싱 실패 시 메모리에 `Settings::default()`를 채우고, 종료 시 `app.save_settings()`가 그 기본값을 무조건 디스크에 씀 → `remote_profiles`(SSH 프로필)·`keybindings`·`theme` 등 영구 삭제. (예: 알 수 없는 키바인딩 액션 키 하나로 전체 파싱 실패)
- **수정**: 로드 실패 여부(`settings_load_failed`)를 캡처해, 실패 시 종료 저장을 건너뜀(원본 보존 우선).

### 3.2 암호화 v3 헤더 버전 다운그레이드 → HMAC 검증 우회
- **파일**: `src/enc/mod.rs` `unpack_file_group`
- **원인**: HMAC 검증이 `header.version >= 3`에서만 수행. 헤더 버전 바이트를 2로 변조하면 무결성 검사가 공격자 제어 필드에 의존. (`MIN_SUPPORTED_VERSION=2` 하위호환과 결합된 약점)
- **수정**: 메타데이터 파싱 후 `meta.version > header.version`이면 다운그레이드로 간주해 거부. 진짜 v2(meta.version=2)는 영향 없음.

### 3.3 `split_size_mb` 무경계 곱셈 오버플로
- **파일**: `src/enc/mod.rs:356`
- **원인**: 사용자 입력 `split_size_mb * 1024 * 1024` — 상한 없는 파싱 값에 곱셈(debug 패닉 / release 랩).
- **수정**: `saturating_mul(1024 * 1024)`.

### 3.4 diff 파일 뷰 무경계 읽기 → 다중 GB 파일 OOM
- **파일**: `src/ui/diff_file_view.rs` `new`
- **원인**: `fs::read`로 양쪽 파일을 통째로 읽고 텍스트·라인·DiffLine으로 4–5배 복제. 크기 가드 없음(`file_viewer`는 100MB 가드 있음).
- **수정**: 100MB 크기 가드 추가, 초과 시 "File too large to diff" 표시.

### 3.5 diff 디렉토리 심링크 순환 → 무한 재귀
- **파일**: `src/ui/diff_screen.rs` `make_file_info`
- **원인**: 심링크 타깃을 `fs::metadata`로 따라가 `is_directory=true`로 표시 → 모든 재귀 경로(빌드/expand-all)가 `ln -s .. loop` 같은 순환에 무한 재귀.
- **수정**: `is_directory = !is_symlink && actual_metadata.is_dir()` — 심링크는 디렉토리로 펼치지 않음(`diff -r` 기본 동작과 동일).

### 3.6 `--ccserver` 토큰 마스킹 바이트 슬라이스 패닉
- **파일**: `src/main.rs:867`
- **원인**: `&token[..8]` — 멀티바이트 문자가 바이트 8을 가로지르면 패닉.
- **수정**: `safe_prefix(token, 8)`(char-boundary 안전).

## 4. 원격 / SSH — 2건

### 4.1 rsync 3.2.4+에서 따옴표 감싸기로 모든 원격 전송 실패
- **파일**: `src/services/remote_transfer.rs` `build_remote_spec`
- **원인**: rsync ≥ 3.2.4는 원격 인자의 특수문자를 자체 이스케이프(modern arg protection)하므로, 코드가 감싼 작은따옴표가 원격에서 **경로의 리터럴 글자**로 취급돼 전송 실패. (로컬 rsync 3.2.7에서 실증: `change_dir "...'..." failed`.) 구버전 rsync에서는 따옴표가 셸 인젝션 방어라 단순 제거 불가.
- **수정**: `rsync --version`을 1회 감지·캐시(`OnceLock`)하여 분기 — 3.2.4+면 따옴표 제거, 미만이면 기존 따옴표 유지. 파싱 실패 시 안전측(따옴표 유지).

### 4.2 `sshpass -p <password>`로 비밀번호가 `ps`/argv 노출
- **파일**: `src/services/remote_transfer.rs`
- **원인**: 패스워드가 프로세스 인자로 전달돼 동일 호스트의 다른 사용자/auditd가 읽을 수 있음.
- **수정**: `sshpass -e` + `SSHPASS` 환경변수로 전환(argv 비노출).

### 검증 후 수정하지 않은 잔여 사항 (보고만)

- **russh 0.46 한계 3건**: (i) known_hosts에 구키+신키가 공존하면 OpenSSH와 달리 KeyChanged로 거부(전송용 ssh CLI는 성공 → 비대칭), (ii) 같은 호스트에 파싱 불가 키 타입(ssh-dss 등) 줄이 있으면 연결 실패, (iii) rsa-sha2-512 단독 협상 서버에서 키 변경 감지가 우회될 수 있는 잠재 경로(현 기본 설정에선 도달 불가). 라이브러리 동작이라 앱 측 수정 범위 밖.
- **file_ops**: 프로세스 크래시 시 `.cokacdir_*` 임시 파일 잔존(시작 시 청소 없음), move 설치 실패 시 진행률 바이트가 미리 집계되는 표시 불일치, 심링크-디렉토리 소스가 file_ops 계층의 자기복사 검사를 우회(UI 경로는 사전 필터로 차단됨, 재귀 깊이 제한으로 폭주 불가).
- **rsync**: 사용자 환경의 `RSYNC_OLD_ARGS=1`은 감지 불가(따옴표 생략과 충돌 가능) — 비주류 환경.
- **enc**: unpack 출력 경로의 존재검사→rename 사이 TOCTOU(로컬 파일관리자 특성상 저위험, 기존 패턴).
- **잔존 stale 임시**: `.{group}.unpacking`이 크래시로 남으면 해당 그룹 unpack이 수동 삭제 전까지 거부(보안상 의도된 동작, 테스트로 고정됨).
